# 本机复现报告 · RTX 4060 Laptop (8GB)

把官方报告里基于 **RTX 3090 (24GB)** 的全部结论,在一台 **8GB 显存的游戏笔记本**上完整复现了一遍,用来回答一个问题:

> **消费级 8GB 笔记本显卡,能不能撑起和 24GB 工作站卡相同的结论?**

**一句话结论:能。** 所有结构性结论(显存固定、长文档不降速、大文件无压力、OCR 精度)在 8GB 卡上逐项复现,显存数字与 3090 几乎逐字节吻合;唯二代价是速度慢约 **1.4–2.5 倍**,以及峰值显存**零余量**(靠 R-SWA 把峰值锁死在 7.54GB 才没 OOM)。

## 1. 显存:R-SWA 固定容量(核心结论,完美复现)

| 阶段 | RTX 3090 报告 | 本机 RTX 4060 |
|:-----|:------------:|:-------------:|
| 加载前空闲显存 | — | 6.94GB |
| 模型加载后占用 | 6.31GB | **6.31GB** ✅ |
| 单页推理峰值 | 7.54GB | **7.53GB** ✅ |
| **100 页文档推理峰值** | 7.54GB | **7.54GB** ✅ |
| **86 页文档推理峰值** | 7.54GB | **7.54GB** ✅ |
| **24MB 大文件推理峰值** | 7.54GB | **7.54GB** ✅ |
| 峰值时空闲显存 | **17.69GB 富余** | **0.00GB(满载压线)** |

**无论 1 页还是 100 页、普通文件还是 24MB 大文件,峰值显存恒定 7.54GB**——这正是官方宣传的 R-SWA(Reference Sliding Window Attention)固定 KV 缓存机制,在 8GB 卡上同样成立。也正因显存被锁死在 7.54GB,这张几乎零余量的卡才能跑完全部测试而不 OOM。

> ⚠️ 本机峰值时空闲显存为 0.00GB,能跑通部分依赖 Windows WDDM 把超出部分溢出到系统内存兜底。3090 则有 17.69GB 富余。这意味着本机一旦再开占显存的程序,就可能真正 OOM。

## 2. OCR 精度 CER(完美复现)

最优参数 `no_repeat_ngram_size=50, ngram_window=128, temperature=0.0`,对照程序化生成挑战图的 ground truth 计算字符错误率。

| 场景 | 报告 CER | **本机 CER** | 编辑距离 / 参考长度 | 判定 |
|:-----|:--------:|:------------:|:-------------------:|:-----|
| c1 中文扫描件 | 0.4% | **0.4%** | 1 / 254 | ⭐ 近乎完美 |
| c5 多栏报纸 | 0.5% | **0.5%** | 3 / 620 | ⭐ 布局完美 |
| c4 低质量扫描 | 26.0% | **26.0%** | 95 / 366 | ✅ 文字 OK |
| c2 手机拍摄 | 重复循环 | 75.0% | 417 / 556 | ❌ 重复 |
| c3 手写体 | 重复循环 | 219.2% | 1460 / 666 | ❌ 重复 |
| c6 盖章合同 | 重复循环 | 227.9% | 1447 / 635 | ❌ 重复 |

精度逐项一致(同权重、temp=0 确定性推理,结果本就该与显卡无关)。c2/c3/c6 的高 CER 正是报告记录的「重复生成」已知局限,在本机同样复现。

## 7. 复现步骤(Windows 原生)

```powershell
# 1. 建环境
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install torch torchvision --index-url https://download.pytorch.org/whl/cu124
.\.venv\Scripts\python.exe -m pip install transformers==4.57.1 pymupdf==1.27.2.2 einops addict easydict matplotlib psutil accelerate huggingface_hub hf_xet

# 2. 关闭 hf_transfer(Windows 上会卡死),改走 Xet
$env:HF_HUB_ENABLE_HF_TRANSFER="0"

# 3. 跑测试(脚本默认读 test_pdfs/<name>.pdf;若 PDF 已移入 arxiv/ 和 challenge/ 子目录,
#    需在 test_pdfs 根目录建同名硬链接,或修改脚本路径)
.\.venv\Scripts\python.exe ocr_challenge.py          # 6 挑战场景
.\.venv\Scripts\python.exe ocr_quantitative_eval.py  # CER 量化
.\.venv\Scripts\python.exe ocr_final_verify.py       # 最优参数验证
.\.venv\Scripts\python.exe ocr_tune.py               # 参数网格
.\.venv\Scripts\python.exe ocr_benchmark_v2.py       # 长文档 + 大文件
```
